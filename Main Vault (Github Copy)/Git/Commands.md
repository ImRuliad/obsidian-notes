
| Command                       | Description                                                            | Use Case                                                      | Example                                                  |
| ----------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------- | -------------------------------------------------------- |
| `git init`                    | Initializes a new Git repository in the current directory              | Starting a brand new project with version control             | `git init`                                               |
| `git clone <url>`             | Creates a local copy of a remote repository                            | Getting a copy of an existing repo to work on                 | `git clone https://github.com/user/repo.git`             |
| `git status`                  | Shows the state of your working directory and staging area             | Checking which files are modified, staged, or untracked       | `git status`                                             |
| `git add <file>`              | Stages a file (or all files) to be included in the next commit         | Preparing specific changes before committing                  | `git add main.py` or `git add .`                         |
| `git commit -m "<msg>"`       | Saves staged changes to the local repository with a message            | Recording a meaningful checkpoint in your work                | `git commit -m "Add login feature"`                      |
| `git push`                    | Uploads local commits to the remote repository                         | Sharing your commits with the team or backing up your work    | `git push origin main`                                   |
| `git pull`                    | Fetches and merges changes from the remote into your current branch    | Syncing your local branch with the latest remote changes      | `git pull origin main`                                   |
| `git fetch`                   | Downloads changes from remote but does NOT merge them                  | Reviewing remote changes before integrating them              | `git fetch origin`                                       |
| `git branch`                  | Lists, creates, or deletes branches                                    | Viewing existing branches or creating a new one               | `git branch feature/login`                               |
| `git checkout <branch>`       | Switches to a different branch                                         | Moving between branches during development                    | `git checkout feature/login`                             |
| `git checkout -b <branch>`    | Creates a new branch and switches to it immediately                    | Starting work on a new feature or fix                         | `git checkout -b feature/signup`                         |
| `git switch <branch>`         | Modern alternative to `checkout` for switching branches                | Switching branches with a cleaner, dedicated command          | `git switch feature/login`                               |
| `git merge <branch>`          | Merges the specified branch into your current branch                   | Integrating a completed feature branch into main              | `git merge feature/login`                                |
| `git rebase <branch>`         | Replays your commits on top of another branch                          | Keeping a clean linear history before merging                 | `git rebase main`                                        |
| `git stash`                   | Temporarily shelves uncommitted changes                                | Quickly switching branches without committing unfinished work | `git stash`                                              |
| `git stash pop`               | Restores the most recently stashed changes                             | Resuming work after switching back to your branch             | `git stash pop`                                          |
| `git log`                     | Shows the commit history for the current branch                        | Reviewing past commits, authors, and messages                 | `git log --oneline`                                      |
| `git diff`                    | Shows changes between your working directory and the last commit       | Reviewing what you've changed before staging                  | `git diff`                                               |
| `git reset <file>`            | Unstages a file without discarding changes                             | Removing a file from staging that was added by mistake        | `git reset main.py`                                      |
| `git reset --hard <hash>`     | Resets the repo to a specific commit, discarding all changes after it  | Reverting to a known good state (use with caution)            | `git reset --hard a1b2c3d`                               |
| `git revert <hash>`           | Creates a new commit that undoes a specific past commit                | Safely undoing a commit without rewriting history             | `git revert a1b2c3d`                                     |
| `git remote -v`               | Lists all remote connections and their URLs                            | Verifying which remotes (origin, upstream) are configured     | `git remote -v`                                          |
| `git remote add <name> <url>` | Adds a new remote repository connection                                | Linking your local repo to a remote for the first time        | `git remote add origin https://github.com/user/repo.git` |
| `git tag <name>`              | Creates a tag at the current commit                                    | Marking a release version in your history                     | `git tag v1.0.0`                                         |
| `git cherry-pick <hash>`      | Applies a specific commit from another branch onto your current branch | Pulling in a single bug fix without merging an entire branch  | `git cherry-pick a1b2c3d`                                |
