
> [!DANGER] PROBLEM STATEMENT
> The **array-form** of an integer `num` is an array representing its digits in left to right order.
> - For example, for `num = 1321`, the array form is `[1,3,2,1]`.
> 
> Given `num`, the **array-form** of an integer, and an integer `k`, return _the **array-form** of the integer_`num + k`.
> 
> **Example 1:**
> - **Input:** num = [1,2,0,0], k = 34
> - **Output:** [1,2,3,4]
> - **Explanation:** 1200 + 34 = 1234
> 
> **Example 2:**
> - **Input:** num = [2,7,4], k = 181
> - **Output:** [4,5,5]
> - **Explanation:** 274 + 181 = 455
> 
> **Example 3:**
> - **Input:** num = [2,1,5], k = 806
> - **Output:** [1,0,2,1]
> - **Explanation:** 215 + 806 = 1021

> [!Info] My Original Thought Process
> My original thought process
> - Cast each element in num to string using `map()`
> - Join the list together into a string using `"".join(num)`
> - Cast the string into an int using `int(...)`
> - Add int `num` to `k` and store it in a result variable
> - Split the result variable by its digits and store it into a list.
> 
> This doesn't work for several reasons...
> - Type conversions are expensive and inefficient
> - Can't convert entire arrays to an integer for digits over ~10,000.
> - Overall just really inefficient.

# <u> Brute Force Code </u>
```python
str_num = "".join(map(str, num))
sum_result = int(str_num) + k
list_result = []
        
while sum_result > 0:
    remainder = sum_result % 10
    list_result.append(remainder)
    sum_result //= 10
            
list_result.reverse()
return list_result
```
